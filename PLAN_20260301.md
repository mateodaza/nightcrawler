## Phase 2: Bash Orchestrator (Days 1-2)

**Goal:** Replace OpenClaw's LLM reasoning with a deterministic bash script. $0 orchestration cost.

### 2.1 Build `nightcrawler.sh`

Single bash script at `~/nightcrawler/scripts/nightcrawler.sh`. Implements the full session lifecycle from SPEC.md §4.1. Calls existing Python scripts — no rewrites of call_opus.py, call_sonnet.py, call_codex.py, budget.py, session.sh, lock_detect.py, or budget_gate.sh.

**Usage:**
```bash
nightcrawler.sh <project> [--budget N] [--dry-run]
# Example: nightcrawler.sh clout --budget 15
```

**Structure:**

```
nightcrawler.sh
│
├── STARTUP (SPEC.md §4.1 steps 1-20, exact order)
│   ├── Generate SESSION_ID: YYYYMMDD-HHMMSS-<project>
│   ├── Crash recovery (session.sh recover — SPEC.md §4.2)
│   ├── Stale lock cleanup (/tmp/nightcrawler-<project>.lock — check PID alive)
│   │   → If lock exists and PID alive → REFUSE, notify, exit
│   ├── Verify base branch (read from openclaw.yaml → projects.<name>.base_branch)
│   │   → If wrong branch → REFUSE, notify, exit
│   ├── Verify clean worktree: git status --porcelain must be empty
│   │   → If dirty → REFUSE, notify with dirty file list, exit
│   │   → NEVER auto-stash. This is a hard startup refusal per RULES.md.
│   ├── Acquire lockfile (session.sh acquire)
│   ├── Switch to dev branch: git checkout nightcrawler/dev || git checkout -b nightcrawler/dev
│   ├── Create session directory + init journal (session.sh)
│   ├── Start heartbeat (session.sh heartbeat-start)
│   ├── Read GLOBAL_PLAN.md, TASK_QUEUE.md, memory.md, PROGRESS.md
│   ├── Reconcile stale [~] In Progress markers → reset to [ ] Queued, log warning
│   ├── Load config (models, budget, rules)
│   ├── Validate Codex availability (call_codex.py --test)
│   │   → If CLI fails, try API → if both fail → REFUSE, release lock, notify, exit
│   │   → This is a SESSION-STOPPING condition per ESCALATION.md §Codex Unavailable
│   ├── BASELINE CHECK: forge build && forge test, record green commit hash
│   │   → If tests fail → notify "repo is red before session", release lock, exit
│   │   → This is a SESSION-STOPPING condition per ESCALATION.md §Pre-Flight
│   ├── Initial dep scan: mark tasks with terminal-failure deps as DEP_BLOCKED
│   ├── Budget init (budget.py init)
│   └── notify_normal "▶️ Session <session-id> started. [N] tasks. Budget: $[X]."
│
├── TASK LOOP (while eligible tasks exist && budget OK)
│   │
│   ├── pick_next_task()
│   │   Re-read TASK_QUEUE.md each iteration (dynamic, not precomputed):
│   │   - Find first [ ] Queued task whose dependencies are ALL satisfied
│   │   - Dependency satisfaction rules:
│   │     • [x] Completed → satisfied
│   │     • [🚧] MANUAL → ignored (TERMS.md: "MANUAL tasks are NOT dependencies")
│   │     • QUEUED or IN_PROGRESS → not yet satisfied, skip this task for now
│   │     • Terminal failure (BLOCKED/SKIPPED/LOCKED) → mark this task DEP_BLOCKED
│   │   - Skip [🚧] MANUAL tasks themselves (orchestrator never attempts them)
│   │   - Return first eligible task ID or "NONE" → SESSION END
│   │
│   ├── BUDGET GATE: effective_remaining = cap - reserve - spent
│   │   → If < $1.00 → stop admitting, proceed to SESSION END
│   │
│   ├── PHASE 0: PRE-FLIGHT CHECK (SPEC.md §4.1 Phase 0)
│   │   ├── Run forge build && forge test
│   │   ├── If pass → proceed to Phase A
│   │   ├── If fail AND were passing at baseline → identify breaking commit
│   │   │   → escalate_urgent "🚨 PRE-FLIGHT FAIL" per ESCALATION.md
│   │   │   → STOP ALL TASKS until resolved (session-stopping, not task-skippable)
│   │   └── If fail AND also failing at baseline → pre-existing, proceed
│   │
│   ├── PHASE A: MINI-PLAN (max 3 iterations — RULES.md §Loop Discipline)
│   │   ├── Extract task spec → temp file
│   │   ├── budget_gate.sh → call_opus.py plan → save mini_plan.md
│   │   ├── budget_gate.sh → call_codex.py audit-plan → JSON output
│   │   ├── Parse verdict: jq -r '.verdict' from JSON (NOT grep on raw text)
│   │   │   → Python scripts return {"verdict": "APPROVED"|"REJECTED", "feedback": "..."}
│   │   ├── If REJECTED:
│   │   │   ├── Store feedback, increment counter
│   │   │   ├── lock_detect.py check (Jaccard on feedback history)
│   │   │   ├── If locked → journal, park task, escalate_urgent per ESCALATION.md §Lock
│   │   │   │   → Park task, continue to next (task-level, not session-stopping)
│   │   │   └── budget_gate.sh → call_opus.py revise → re-audit
│   │   └── If APPROVED → proceed to Phase B
│   │
│   ├── PHASE B: IMPLEMENT (max 3 iterations)
│   │   ├── budget_gate.sh → call_sonnet.py implement
│   │   ├── forge build && forge test (verify before sending to audit)
│   │   ├── budget_gate.sh → call_codex.py review-impl → JSON output
│   │   ├── Parse verdict: jq -r '.verdict' from JSON
│   │   ├── If REJECTED:
│   │   │   ├── Same lock detection pattern
│   │   │   └── budget_gate.sh → call_sonnet.py revise → re-review
│   │   └── If APPROVED → proceed to Phase C
│   │
│   ├── PHASE C: CLOSE TASK (atomic — SPEC.md §4.1 Phase C)
│   │   ├── journal: {"event": "committing", "task_id": "NC-XXX"}
│   │   ├── Stage all changes: git add -A
│   │   ├── Commit with structured message per SPEC.md §7.3
│   │   │   → Include: task code + PROGRESS.md + TASK_QUEUE.md updates in ONE commit
│   │   │   → No separate "progress update" commit. One atomic close.
│   │   ├── POST-COMMIT VERIFY: forge build && forge test (full suite)
│   │   ├── If FAIL:
│   │   │   ├── git revert HEAD --no-edit
│   │   │   ├── journal: {"event": "reverted", "reason": "..."}
│   │   │   ├── Re-enter Phase B with integration error as feedback
│   │   │   └── If 2nd revert same task → LOCK, escalate per ESCALATION.md §2x Revert
│   │   │       → Park task, continue (task-level)
│   │   ├── journal: {"event": "task_completed", "task_id": "NC-XXX", "commit": "<hash>"}
│   │   ├── notify_normal "✅ NC-XXX completed. Remaining: N. Spent: $X/$Y."
│   │   └── Budget check → if effective_remaining < $1 → exit loop
│   │
│   └── Next task or exit loop
│
├── SESSION END (always runs, even on crash — uses $2 reserve)
│   ├── journal: {"event": "session_ending", "reason": "..."}
│   ├── Generate report → sessions/<session-id>/report.md
│   ├── Update nightcrawler/memory.md with session learnings
│   ├── journal + fsync: {"event": "session_complete"} ← MUST be durable before cleanup
│   ├── notify_normal "⏹️ Session <session-id> done. N completed. Report ready."
│   ├── session.sh release (lockfile)
│   └── session.sh heartbeat-stop
│
├── MESSAGING FUNCTIONS (two tiers, per ESCALATION.md §Rate Limits)
│   │
│   ├── notify_normal() — non-blocking notifications
│   │   ├── openclaw message send --channel telegram (or configured channel)
│   │   ├── Rate-limited: 5/hour, 15-min batch window
│   │   ├── If delivery fails → write to $SESSION_DIR/notifications.log
│   │   │   → Acceptable: these are informational, not safety-critical
│   │   └── Duplicate coalescing: one notification per event per task
│   │
│   ├── escalate_urgent() — blocking escalations (URGENT priority)
│   │   ├── openclaw message send --channel telegram --priority urgent
│   │   ├── NO rate limiting, NO batching — always immediate
│   │   ├── If delivery fails:
│   │   │   ├── Retry once after 30s
│   │   │   ├── If still fails → THIS IS A SESSION-STOPPING FAULT
│   │   │   ├── Log: "URGENT escalation delivery failed, cannot proceed unattended"
│   │   │   └── Proceed to SESSION END (unattended safety model is broken)
│   │   ├── After delivery: poll for response per ESCALATION.md response parsers
│   │   └── Timeout behavior is TYPE-SPECIFIC (see below)
│   │
│   └── Escalation timeout behavior (NOT one-size-fits-all):
│       │
│       ├── TASK-LOCAL (park task, continue to next):
│       │   - Lock Detected → park, move to next task
│       │   - Ambiguous Task → skip, move to next task
│       │   - Test Failures Post-Approval → park, move to next task
│       │   - 2x Revert → park, move to next task
│       │   Timeout: 2 hours, then auto-park
│       │
│       └── SESSION-STOPPING (halt the session):
│           - Budget Exceeded → SESSION END (uses $2 reserve)
│           - Codex Unavailable → SESSION END (independent audit non-negotiable)
│           - Pre-Flight Failure → STOP ALL TASKS (previous commit broke something)
│           - Urgent delivery failure → SESSION END (safety model broken)
│           These NEVER auto-skip. They wait indefinitely or end the session.
│
├── OTHER HELPER FUNCTIONS
│   ├── log()        — timestamped append to session log
│   ├── journal()    — session.sh journal wrapper
│   └── die()        — log error, attempt SESSION END, exit
│
└── SIGNAL HANDLERS
    ├── SIGINT/SIGTERM → graceful SESSION END
    ├── ERR → log, attempt SESSION END
    └── EXIT → cleanup temp files
```

## Phase 3: Ship Clout Tasks (Days 2-8)

**Goal:** Execute remaining queued tasks. This is the point of everything.

### 3.1 Session strategy

Task counts and session chunks come from the current TASK_QUEUE.md at session start — not hardcoded here. The orchestrator reads the queue dynamically (SPEC.md §4.1 PICK NEXT TASK).

Budget planning starts from Phase 1.2 measurement but is refined over time:
- **First session:** use measured cost × target task count × 1.5 safety multiplier (single sample is weak — later tasks with more revisions/integration will cost more)
- **After 3-5 real tasks:** switch to p75 of observed task costs × expected tasks + $2 reserve
- Tighten the multiplier as data accumulates; never plan with just the mean
- If measured cost is ~$0.20/task total: $3/session is conservative enough for first run
- If measured cost is higher: reduce task target per session rather than raising budget
- MANUAL tasks ([🚧]) are auto-skipped per TERMS.md
- Gate tasks (NC-G1, NC-G2, etc.) are real dependency barriers — they stay in the dependency graph and block downstream work until completed. They are NOT auto-skipped.

### 3.2 Morning review protocol

After each overnight session:
1. `cat ~/nightcrawler/sessions/<latest-session-id>/report.md`
2. Review commits: `cd /home/nightcrawler/projects/clout && git log --oneline nightcrawler/<session-id>`
   (Use the actual session-id from the report, not a hardcoded branch name)
3. Run full suite: `forge build && forge test`
4. Check for skipped/locked/blocked tasks and why
5. Fix blockers, adjust TASK_QUEUE.md if needed
6. Start next session

### 3.3 Manual tasks

Tasks marked [🚧] MANUAL are auto-skipped by the orchestrator (TERMS.md §MANUAL).
Schedule them after automated tasks are done.

## Phase 4: Hardening + Quality (Days 4-7, parallel with Phase 3)

**Goal:** Make the pipeline more reliable and the output more secure.

### 4.1 Strengthen Codex audit prompts

Add Solidity-specific checklist to call_codex.py audit prompts:
- nonReentrant on ALL external state-changing functions
- Events emitted on ALL state transitions
- CEI pattern (checks-effects-interactions) on all external calls
- No unbounded loops over dynamic arrays
- All amounts use 6 decimals (stablecoin native)
- No test patching (if test expectations changed, verify they're correct)
- Access control: no onlyOwner on view functions, missing modifiers on state-changing ones
